  • canvas

    does not persist; always rerendered; fire and forget; pixel manipulation; drawing fast, and not needing to retain this information; high performance for filters and ray traces; real-time data output (maps for instance) video manipualtion (eg. greenscreen for color detection and replacing background colors) low level pixel basic (dynamic png); single html element; modified through script only event model is granular (x, y) Performance is worse with a big screen size (many pixels to render); number of objects has no considerable impact

  • svg

    retained mode graphics model, used in HTML5, which persists in an in-memory model that can be manipulated through code results in re-rendering. shape based; multiple graphical elements, which become part of the DOM; modified through script and CSS event model is abstracted (rect, path) Performance is worse with a great number of objects; screen size has no considerable impact High fidelity and scalable complex vector documents

    CSS Attributes

    • transform

      values: none (default), transform function (matrix, perspective, rotate, translate, scale, skew) description: It modifies the coordinate space of the CSS visual formatting model to let you rotate, scale, skew, or translate an element reference: https://developer.mozilla.org/en-US/docs/Web/CSS/transform

    • float

      values: none (default), right, left description: laces an element on the left or right side of its container, allowing text and inline elements to wrap around it. The element is removed from the normal flow of the page, though still remaining a part of the flow (in contrast to absolute positioning).; side-effect: As float implies the use of the block layout, it modifies the computed value of the display values

    • box-sizing: content-box (default), border-box - determines where borders are included or not in width and height properties

    • text-transform

      none (default), uppercase, lowercase, capitalize description: Transform text in different div elements: The border-collapse property sets whether table borders should collapse into a single border (border sharing) or be separated as in standard HTML (separate borders). values: collapse (standard), separate Eg.: By border-collpase: collapse, if there's one div on top of the other in the document flow, the one with greather margin value overwrittes the other. Still, this default behavior can be prevented by setting border-collapse: separate or setting any border (border: 2x solid black)

    • text-decoration

      default: text-decoration-color: currentcolor text-decoration-style: solid text-decoration-line: none description: sets the appearance of decorative lines on text. eg. text-decoration: underline overline #FF3028; reference: https://developer.mozilla.org/en-US/docs/Web/CSS/text-decoration

    CSS Concepts

    • Border Colapsing

      The border-collapse property sets whether table borders should collapse into a single border (border sharing) or be separated as in standard HTML (separate borders). values: collapse (standard), separate Eg.: By border-collpase: collapse, if there's one div on top of the other in the document flow, the one with greather margin value overwrittes the other. Still, this default behavior can be prevented by setting border-collapse: separate or setting any border (border: 2x solid black)

    • Grid

      Normally, we have a wrapper with display: grid and the template values (eg. grid-template-columns: repeat(3, 1fr); gap: 10px; grid-auto-rows: minmax(100px, auto);) After that, the grid child elements are positioned in the grid field, within the range 1, max-rows/columns + 1 eg.: 1) grid-column: 1 / 3; grid-row: 1; 2) grid-column: 1 / span 2; grid-row: 1 / 2; (exactly the same result)

    • CanvasGradient

      The CanvasGradient interface represents an opaque object describing a gradient. It is returned by the methods CanvasRenderingContext2D.createLinearGradient() or CanvasRenderingContext2D.createRadialGradient(). CanvasGradient.addColorStop() Adds a new stop, defined by an offset and a color, to the gradient. If the offset is not between 0 and 1, inclusive, an INDEX_SIZE_ERR is raised; if the color can't be parsed as a CSS , a SYNTAX_ERR is raised.
